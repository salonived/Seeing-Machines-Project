"Tracking the systematic gaps between text-query modifiers and visual embedding spaces."
Abstract
This study explores how a visual-language search model interprets a personal dataset. Using a search portal constructed with Gradio, I evaluated how shifting similarity thresholds altered the cross-modal alignment profiles of returned files. The results demonstrate how literal token semantics often pull embedding matching focus away from layout structures and design motifs. This documentation traces those systematic operations, capturing where human intent and machine computations align or reveal errors.
Motivation & Corpus Statement
My Personal Corpus (71 Images):
My corpus contains 71 original images from my creative practice as an illustrator and branding designer, spanning portfolio work, poster designs, and sketched illustrations created over several years. Because I generated every single asset myself, I possess complete context regarding the specific design briefs, style flags, color choices, and composition grids intended. This intimate knowledge acts as a rigorous baseline to evaluate machine performance, enabling me to immediately detect when a retrieval match occurs due to surface-level pixel distributions rather than genuine conceptual relationships.
Approach & Setup Pipeline

The code extracts high-dimensional visual vectors from static image files. When an active query string is submitted, the system computes a matching text embedding and calculates nearest neighbors using normalized cosine similarity scores.

def lookup_engine(text_input, total_k, score_minimum):
    text_vector = model.encode_text(clip.tokenize([text_input])).to(device)
    text_vector /= text_vector.norm(dim=-1, keepdim=True)
    
    match_scores = (image_matrix @ text_vector.T).squeeze(1).tolist()
    sorted_pairs = sorted(zip(all_filenames, match_scores), key=lambda x: x[1], reverse=True)
    
    return [item for item in sorted_pairs if item[1] >= score_minimum][:total_k]

Critical Systemic Synthesis
Comparing these 15 evaluation scenarios reveals an important performance truth: even when a specific portfolio item surfaces as the top match across completely different prompts, its absolute similarity score stays low (often hovering around 0.078). This proves the engine is not confident it found an exact match; it is simply rendering the least distant point available in a closed space where a perfect match does not exist.
Project Reflection & Dead Ends
Building this setup showed me how easily we can misunderstand machine sorting. In my initial steps, I tried to implement code to shift the score threshold dynamically whenever searches returned empty grids. This caused bugs where non-relevant design sheets crowded the layout view. Realizing that human alignment adjustments matter more than forcing neat automated matches changed how I approach data spaces.
Limitations
The primary limit comes from file downsampling within the vector model. Subtle layout changes, fine textures, and fine text lines lose definition, causing the program to focus heavily on main color fields and simple silhouettes.
