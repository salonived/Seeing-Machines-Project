"Tracking the gaps between prompt text modifiers and spatial image matching layout limits."
Abstract
This study explores how a visual-language search model interprets a personal dataset. Using a search portal made with Gradio, I evaluated how changing the similarity settings changed what files were returned. The results show that basic item names often pull the search focus away from layout and design styles. This file serves as a complete map of those interactions, capturing where human ideas and machine calculations work together or miss each other entirely.
Motivation & Corpus Statement
My Personal Corpus (71 Images):
My library is built from 71 original vector images, branding projects, layouts, and sketch concepts created throughout my design journey. Because I created every file myself, I know exactly what style decisions, colors, and layout compositions were intended. This makes it easy to spot whenever the retrieval model mixes up or mis-sees an item based only on surface similarities instead of its deeper design structure.
Approach & Setup Pipeline

The code turns static image assets into high-dimensional vectors. When a user enters a query string, the system generates a corresponding text embedding vector and calculates the closest matches using cosine similarity scores.

def lookup_engine(text_input, total_k, score_minimum):
    text_vector = model.encode_text(clip.tokenize([text_input])).to(device)
    text_vector /= text_vector.norm(dim=-1, keepdim=True)
    
    match_scores = (image_matrix @ text_vector.T).squeeze(1).tolist()
    sorted_pairs = sorted(zip(all_filenames, match_scores), key=lambda x: x[1], reverse=True)
    
    return [item for item in sorted_pairs if item[1] >= score_minimum][:total_k]

Critical Systemic Synthesis
Comparing these 13 scenarios reveals an important performance truth: even when a specific portfolio item surfaces as the top match across completely different prompts, its absolute similarity score stays low (hovering around 0.078). This proves the engine is not confident it found an exact match; it is simply rendering the least distant point available in a closed space where a perfect match does not exist.
Project Reflection & Dead Ends
Building this setup showed me how easily we can misunderstand machine sorting. In my initial steps, I tried to add automated code to shift the score threshold dynamically whenever searches returned empty grids. This caused bugs where non-relevant design sheets crowded the layout view. Realizing that human alignment adjustments matter more than forcing neat automated matches changed how I approach data spaces.
Limitations
The primary limit comes from file downsampling within the vector model. Subtle layout changes, fine textures, and fine text lines lose definition, causing the program to focus heavily on main color fields and simple silhouettes.
