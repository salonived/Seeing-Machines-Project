Model Engine: SigLIP / CLIP Multimodal
"Tracking the systematic gaps between text-query modifiers and visual embedding spaces."
Abstract
This project explores how a multimodal vision-language model interprets a highly specific, personal archive of creative work. Using a custom search interface built with Gradio, I tested a semantic image retrieval system on 71 artworks from my own portfolio, documenting exactly how the machine "sees" through visual embeddings. The core finding: SigLIP excels at matching explicit visual grammar (colors, textures, styles) but fundamentally struggles with abstract reasoning, entity recognition, and implicit visual concepts.
Motivation & Corpus Statement

The corpus consists of 71 images selected from my personal practice as an illustrator and graphic designer. It includes portfolio work, brand identity assets, poster layouts, and digital illustrations created over three years of independent practice.

Because every image in this archive is a piece of work I created myself, I know the exact intent behind it: the specific design brief, the deliberate color choices, the conceptual reasoning. This knowledge allows me to recognize what the model "misses"—not failures of data, but failures of understanding at a more fundamental level. When a query returns incorrect results, I can diagnose whether the issue is vocabulary, spatial reasoning, or something else entirely.

Approach & Pipeline
The system runs on a retrieval pipeline that maps both text and images into a shared mathematical space:
Image Embedding Generation: The 71 original portfolio images are processed through a vision-language model (SigLIP framework), converting their visual features into high-dimensional vectors (typically 768-1024 dimensions). These vectors are cached for efficient retrieval.
Text Embedding Generation: When a user inputs a search query into the web interface, the system translates the text string into a matching text vector within the same embedding space using the model's text encoder.
Similarity Metrics: The retrieval engine calculates the closeness (cosine similarity) between the text vector and all cached image vectors, returning ranked results above a configurable confidence threshold.
Interface Control: A custom Gradio interface displays the top matching images in a clean grid, allowing manual adjustment of the similarity threshold and maximum number of results returned per query. 
def lookup_engine(text_input, total_k, score_minimum):

Critical Systemic Synthesis
Comparing these 13 evaluation scenarios reveals an important performance truth: the model excels when visual features align with text descriptors, but collapses when asked to reason abstractly about composition, semantics, or concepts that require knowledge beyond visual grammar.
Limitations

Entity Knowledge Gaps

The model struggled with specific entity recognition (e.g., 'Taylor Swift', 'Shah Rukh Khan'), often defaulting to color or mood proxies instead of actual features. When a named entity is absent from the training data or the archive lacks obvious visual markers, the system fails to construct a meaningful match and instead returns arbitrary results based on whatever low-level features it can isolate.

Composition Bias

For queries like 'Cows falling', the model sometimes prioritized literal compositional matches (e.g., blue/white vertical layouts) over the intended conceptual link. This indicates a 'mis-seeing' due to compositional similarities rather than semantic understanding—the model is shape-matching against archive elements that happen to share spatial or color characteristics without grasping the narrative intent.

Out-of-Distribution (OOD) Queries

Queries like 'Blue car' led to over-indexing on dominant tokens (e.g., 'blue') even when the core subject ('car') was absent from the archive. This highlights a critical challenge in handling queries where the core subject is outside the corpus—the model defaults to whatever features it can extract from the query text and desperately searches for matches in the embedding space, often sacrificing semantic precision for any signal at all.

Fine-Grained Detail Loss

The primary limit comes from file downsampling within the vector model itself. Subtle layout changes, fine textures, and fine text lines lose definition during the encoding process, causing the system to focus heavily on macro-level features (overall color distribution, high-level composition) while ignoring nuanced visual details that humans immediately perceive.

Project Reflection

[1. What Worked: Technical & Stylistic Alignment]

Why: SigLIP is trained on massive volumes of web data including design blogs, portfolio sites, and creative industry content.

Visual Grammar: Terms like 'flat vector', 'typography', 'macro', and 'ornate' are highly effective because they describe image structure rather than just objects. The model has learned to recognize these design idioms across thousands of examples.

Style Mapping: The model successfully identified project-specific aesthetics because the query descriptions matched the technical execution of the artwork. When I described the visual properties accurately ('bold, high-contrast modern typography'), the embeddings aligned perfectly.

[2. What Did Not Work: Semantic & Knowledge Gaps]

Why: These are 'mis-seeings' where the model forces the archive to fit concepts it doesn't contain, or where the query demands reasoning beyond visual matching.

Entity Knowledge: 'Taylor Swift' or 'Shah Rukh Khan' failed as literal queries because the model looks for specific human facial features not present in my illustrations. It defaults to color/mood proxies instead—a fallback strategy that rarely produces meaningful results when the core entity is genuinely absent.

Literal vs. Compositional: 'Cows falling' failed as a literal search but 'succeeded' as a compositional match, surfacing my blue/white vertical layouts. This shows a Composition Bias—the model conflates spatial arrangement with semantic content, treating 'falling' as a layout descriptor rather than an action.

Out-of-Distribution (OOD) Handling: Queries like 'Blue car' over-indexed on 'blue' because the model is desperate to find a match for the dominant token even if the noun is completely missing. It's a form of linguistic triage where the model sacrifices specificity to return something.

Why This Project Matters

This research successfully demonstrated the power of SigLIP for image retrieval within a personalized archive. The model showed strong capabilities in aligning with visual grammar and stylistic intents, effectively identifying project-specific aesthetics by matching descriptive queries to the technical execution of the artwork. This suggests that vision-language models can be valuable tools for organizing and searching creative portfolios—but only when queries focus on visual characteristics and artistic styles, not abstract reasoning or entity knowledge.

The project also revealed something more important: that understanding where and why a machine "mis-sees" teaches us more about both the tool and ourselves than knowing where it succeeds. By documenting these failures methodically, I've created a map of the gap between human visual understanding and mathematical similarity spaces.
