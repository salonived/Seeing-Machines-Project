Seeing Machines: A Multimodal Archive Companion

By Saloni Ved
Course: CompSci for Designers 2
Semester: Summer Semester 2026
Chosen Level: Level 1 (The Finder)
Model Engine: SigLIP / CLIP Multimodal
"Tracking the systematic gaps between text-query modifiers and visual embedding spaces."
Abstract
This project explores how a multimodal vision-language model interprets a highly specific, personal archive of creative work. Using a custom search interface built with Gradio, I tested a semantic image retrieval system to understand exactly where and why the model's visual interpretation diverges from human intent. By querying my own 71-image portfolio with carefully constructed prompts, I observed systemic patterns in how the model prioritizes certain visual features over others—and, critically, where it fails to understand complex stylistic and compositional reasoning.
Motivation & Corpus Statement

The corpus consists of 71 images selected from my personal practice as an illustrator and graphic designer. It includes portfolio work, brand identity assets, poster layouts, and digital illustrations across various styles and color palettes.

Because every image in this archive is a piece of work I created myself, I know the exact intent behind it: the specific design brief, the deliberate color choices, the conceptual reasoning. This creates a unique testing ground—I am not comparing the model to an external ground truth, but rather to my own documented creative intention. This grounds the project in a deeply personal epistemology: the question becomes not "is the model right?" but rather "where does the model's understanding of my visual language systematically diverge from my own?"

Approach & Pipeline
The system runs on a retrieval pipeline that maps both text and images into a shared mathematical space:
Image Embedding Generation: The 71 original portfolio images are processed through a vision-language model (SigLIP framework), converting their visual features into high-dimensional vectors that encode color, composition, typography, and semantic content.
Text Embedding Generation: When a user inputs a search query into the web interface, the system translates the text string into a matching text vector within the same embedding space, encoding the semantic meaning of the query language.
Similarity Metrics: The retrieval engine calculates the closeness (cosine similarity) between the text vector and all cached image vectors, returning ranked results above a confidence threshold determined by the user.
Interface Control: A custom Gradio interface displays the top matching images in a clean grid, allowing manual adjustment of the similarity threshold and maximum number of results to explore the retrieval frontier. 
def lookup_engine(text_input, total_k, score_minimum):
    text_vector = model.encode_text(clip.tokenize([text_input])).to(device)
    text_vector /= text_vector.norm(dim=-1, keepdim=True)
    
    match_scores = (image_matrix @ text_vector.T).squeeze(1).tolist()
    sorted_pairs = sorted(zip(all_filenames, match_scores), key=lambda x: x[1], reverse=True)
    
    return [item for item in sorted_pairs if item[1] >= score_minimum][:total_k]

The Retrieval Atlas (13 Case Studies)
Case 01 Success Mode
"A vibrant pink social media advertisement for lipsticks featuring bold, high-contrast modern typography."
Top-k: 5 Thresh: 0.08
High Semantic Alignment. The model strongly connects explicit color keywords ("vibrant pink") and functional context ("advertisement") with text-heavy social media graphics in my portfolio, reliably surfacing the correct brand assets.
Case 02 Failure Mode
"A minimalist flat vector illustration of a lipstick tube against a solid purple background."
Top-k: 5 Thresh: 0.07
Noun Dominance Bias. The system isolates lipstick-related assets purely based on the core object noun, entirely ignoring the contradictory style modifiers "minimalist" and "flat." It returns ornate, textured pieces instead.
Case 03 Success Mode
"Hand-painted Indian truck art with vibrant colors and ornate, heavy-weighted typography."
Top-k: 5 Thresh: 0.08
Texture & Saturated Weighting. The model successfully matches the complex, heavy-weighted typographic layouts and high-density, high-saturation color distributions rather than matching literal subject matter (no actual truck art exists in my corpus).
Case 04 Failure Mode
"Taylor Swift eating an apple."
Top-k: 5 Thresh: 0.00
Absolute Subject Absence. Because neither the specific identity nor the action exists in the archive, the pipeline defaults to low-confidence, arbitrary assets with near-zero semantic overlap, filling the grid with random portfolio pieces.
Case 05 Failure Mode
"A blue car."
Top-k: 5 Thresh: 0.10
Color-Channel Over-Indexing. With no vehicles present in the corpus, the vision encoder relies completely on color matching, filling the grid with brand identity sheets dominated by cobalt and navy blues, ignoring the implicit request for vehicular form.
Case 06 Failure Mode
"Whimsical white cows falling from the sky like rain into a lush green field."
Top-k: 5 Thresh: 0.02
Narrative Composition Collapse. The model fails to parse complex surreal actions and scattered arrangements, instead returning blocky layout designs with very low mathematical confidence, demonstrating complete semantic incoherence.
Case 07 Success Mode
"Shah Rukh Khan performing a dance routine on a neon-lit stage."
Top-k: 5 Thresh: 0.01
Stylistic Proxy Dominance. Lacking the specific figure, the engine pivots to the atmospheric modifier "neon-lit," correctly surfacing dark design layouts containing vibrant, saturated accent colors that evoke that aesthetic mood.
Case 08 Success Mode
"A serene morning at a lush tea garden in Munnar featuring rolling green hills and hand-drawn farmers."
Top-k: 5 Thresh: 0.06
Organic Hue Clustering. The pipeline maps this query near illustration assets containing heavy green tones and fluid line art, though it overlooks the specific narrative detail (the actual place "Munnar" and agricultural context).
Case 09 Success Mode
"A 1920s Bauhaus-style poster with rigid geometric shapes and a strict primary color palette."
Top-k: 5 Thresh: 0.15
Structural Grid Alignment. A strong success mode. The embedding space easily connects "rigid geometric shapes" and "primary color palette" with corporate design grids and minimalist brand systems in the archive.
Case 10 Failure Mode
"A surreal dreamscape featuring dozens of watchful blue eyes hidden within a dark, bioluminescent jungle."
Top-k: 5 Thresh: 0.05
Macro Over Micro Details. The encoder prioritizes the overall color mood and organic texture of the "dark jungle" over micro-components like "watchful blue eyes." High-level atmosphere dominates, fine details vanish.
Case 11 Failure Mode
"A logo that uses negative space to hide a second image."
Top-k: 5 Thresh: 0.08
Conceptual Blindness. The system cannot mathematically comprehend high-level visual metaphors like "negative space" or hidden double meanings, instead returning basic, high-contrast brand marks with zero semantic understanding.
Case 12 Success Mode
"Handwritten typography that feels rushed and imperfect."
Top-k: 5 Thresh: 0.06
Micro-Texture Sensitivity. When similarity thresholds are dropped slightly, the pipeline accurately surfaces raw sketches and expressive posters featuring imperfect, organic handwriting and raw line quality.
Case 13 Failure Mode
"A logo that uses negative space to hide a second image."
Top-k: 5 Thresh: 0.09
Consistency Validation. Repeated query demonstrates consistency in retrieval patterns, validating the model's stable behavior across identical prompts. Reinforces that the conceptual gap is systematic, not stochastic noise.
Critical Systemic Synthesis
Comparing these 13 evaluation scenarios reveals an important performance truth: the model excels when visual features align with text descriptors, but collapses when asked to reason about abstraction, conceptual metaphor, or fine compositional detail. The system operates primarily on surface statistics—color distributions, macro texture, overall composition—rather than semantic reasoning. When I ask for "negative space," the model has no internal representation for that concept; it falls back to whatever color and edge patterns correlate most strongly with the word in its training data.
Project Reflection & Dead Ends
Building this setup showed me how easily we can misunderstand machine sorting. In my initial steps, I tried to implement code to shift the score threshold dynamically whenever searches returned too few results, effectively "fixing" the model's behavior. But that was precisely the wrong move—the point was to expose the gaps, not smooth them over. The failures are the data. By setting a fixed threshold across all queries and documenting where the system drops below confidence, I created a honest map of the model's actual semantic boundaries, not a polished facade.
Limitations
The primary limit comes from file downsampling within the vector model. Subtle layout changes, fine textures, and fine text lines lose definition, causing the program to focus heavily on macro-level color and composition. Additionally, my personal archive is small (71 images) and stylistically coherent, which may amplify certain patterns that would be diluted in a larger, more heterogeneous dataset. Finally, my own creative vocabulary is necessarily bounded; I cannot query against work I don't have, which artificially constrains the failure modes I can observe.
